# 最小限のベクトル検索アプリを構築する

## 概要

AIは現在最先端の分野であり、多くの開発者が学びたいと考えています。しかし、学習プロセスには以下のような障壁が存在します：

- AIの基礎知識の不足
- 複雑すぎる基盤インフラストラクチャ
- AIアプリケーションの包括的なステップバイステップのドキュメントの欠如

これらの問題に対処するため、このラボを開発しました。このラボを通じて、以下のことを学ぶことができます：

- AIアプリケーションのデータストレージ層として**TiDB Serverless**を使用し、AI開発プロセスを加速する方法
- エンベディングとは何か、そしてその機能
- エンベディングベクトルを取得した後の活用方法
- 大量のデータを扱う際にRAG/GraphRAGを使用してデータを検索する方法
- **DSPy**を使用してテキストからグラフを抽出し、これらのグラフをTiDB Serverlessに保存し、その後グラフ検索を使用してRAG機能を強化する方法

### タスク

- **テキスト類似性検索**

  - OpenAIのエンベディングモデルとAPIを使用してテキストのエンベディングを行う
  - エンベディングベクトルと元のテキストをTiDB Serverlessに保存する
  - TiDBのSQLを使用してベクトル類似性検索を実行し、ユーザークエリに最も関連性の高いテキストデータを取得する

- **画像検索**

  - CLIP事前学習モデルを使用して画像の特徴抽出を行い、エンベディングを取得する
  - エンベディングベクトルと元の画像IDをTiDB Serverlessに保存する
  - TiDBのSQLを使用してベクトル類似性検索を実行し、与えられた画像と特徴が最も類似している画像を取得する

- **GraphRAG検索**

  - TiDB Serverlessに事前構築されたグラフで検索を実行する
  - グラフを使用してユーザークエリに最も関連性の高いテキストデータを取得する

- **Build to Read: 完全なGraphRAGチュートリアル**

  - DSPyを使用してテキストからエンティティと関係を抽出し、グラフを作成する
  - エンティティに対してエンベディングベクトル化を実行する
  - エンティティと関係をTiDB Serverlessに保存する
  - TiDB Serverless内でグラフをクエリしてRAGのパフォーマンスを向上させる

### 所要時間

- このラボは約1.5時間で完了する予定です。
